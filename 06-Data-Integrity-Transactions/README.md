# 06 - Data Integrity and Transactions
Ensuring data consistency, safety, and atomicity during modification operations.

### Key Skills Demonstrated:
* **Transaction Management:** Managing `BEGIN`, `COMMIT`, and `ROLLBACK` for safe updates.
* **Error Handling:** Implementing robust `TRY...CATCH` blocks for database stability.
* **Constraint Auditing:** Validating data against `FOREIGN KEY` and `CHECK` constraints.

### Directory Contents:
1. **01-Safe-Price-Update-Transaction.sql** – Bulk price updates with automatic rollback on error.
2. **02-Audit-Log-Trigger-Implementation.sql** – Tracking changes to sensitive tables using triggers.
3. **03-Data-Cleanup-Constraint-Check.sql** – Script to identify and fix orphaned records.
4. **04-Bulk-Insert-Validation.sql** – Handling data integrity during large-scale imports.
